# Unified Corrections Dashboard

**Date:** 2026-04-04  
**Status:** Approved

## Goal

Replace the multi-step feedback loop (prepare → annotate → finalize → train) with a single, cohesive web interface that allows users to queue all files (original samples + inference results), review predictions, make corrections, and trigger retraining — all from one dashboard.

## Context

Current workflow is fragmented:
- `feedback.py prepare` — select images with misses
- `annotate.py --mode=correct` — review predictions
- `feedback.py finalize` — move corrections
- `train.py` — retrain

Users want one unified interface where:
- All files appear in a persistent queue
- Metadata is visible (status, source, confidence, timestamps)
- All corrections happen in one place
- Training is triggered manually with a button

## Architecture

**Single HTTP server** (`corrections_dashboard.py`) that:
- Maintains persistent queue state (corrections_queue.json)
- Auto-discovers files from photo_mapping_samples/ (originals) and infer_output/ (inference results)
- Loads pre-computed predictions from corrections_meta.json (generated by infer.py batch run)
- Serves browser-based queue + canvas editor UI
- Handles all file I/O, metadata tracking, and label saving
- Triggers train.py on demand via "Train Now" button

## Queue Data Structure

### corrections_queue.json

```json
{
  "files": [
    {
      "stem": "00001500",
      "source": "original" | "inference",
      "status": "pending" | "reviewed" | "needs_fix",
      "created_at": "2026-04-03T10:00:00",
      "last_reviewed_at": "2026-04-03T14:30:00",
      "inference_data": {
        "x": 0.85,
        "y": 0.92,
        "w": 0.18,
        "h": 0.06,
        "confidence": 0.42
      },
      "user_correction": {
        "x": 0.85,
        "y": 0.93,
        "w": 0.18,
        "h": 0.06,
        "action": "confirmed" | "edited" | "skipped" | "no_stamp"
      }
    }
  ],
  "last_inference_run": "2026-04-03T09:00:00"
}
```

### Fields

- **stem**: Filename without extension (e.g., "00001500")
- **source**: "original" (from photo_mapping_samples/) or "inference" (from infer_output/)
- **status**: 
  - "pending" — not yet reviewed
  - "reviewed" — user has taken an action
  - "needs_fix" — marked for re-inference (skipped files)
- **created_at**: When file entered queue
- **last_reviewed_at**: Timestamp of last user action
- **inference_data**: Model's bounding box prediction (only for inference files, null for originals)
- **user_correction**: User's action and final box coordinates
  - "confirmed" — user accepted model prediction as-is
  - "edited" — user adjusted the box
  - "skipped" — deferred, will re-infer next session
  - "no_stamp" — user confirmed no date stamp (negative example)

## UI & Workflow

### Layout (Three Sections)

**Left Sidebar: Queue View**
- Scrollable list of all files with metadata cards
- Each card shows:
  - Thumbnail
  - Status badge (Pending/Reviewed/Needs Fix)
  - Source tag (Original vs. Inference)
  - Confidence % (if inference)
  - "Last reviewed: X days ago"
  - Date added
- Filter buttons: "All / Pending / Reviewed / Needs Fix"
- Sort options: "Date Added / Last Reviewed / Confidence"
- Click any file to open in editor

**Center: Canvas Editor**
- Large image display
- **Yellow box** = model prediction (inference files only)
- **Gray box** = original file (no prediction)
- **Green box** = user adjustment (if edited)
- Keyboard controls:
  - Arrow keys: Move box (20px steps)
  - Shift + arrows: Fine move (3px steps)
  - `[` / `]`: Shrink/grow width
  - `{` / `}`: Shrink/grow height
  - Enter: Confirm and move to next
  - S: Mark no stamp / skip
  - Z: Back to previous
  - Q: Quit (saves state)
- Status bar: filename, queue position (X/Y), source, confidence

**Right Sidebar: Action Panel**
- Buttons: "Confirm / Edit Mode / Mark No Stamp / Skip / Next"
- Current box coordinates display (normalized, editable)
- Confidence display (if inference)
- "Train Now" button (bottom, disabled while training)

### Workflow

1. **Queue view:** User sees all pending files, filters/sorts as needed
2. **Select file:** Click to open in canvas editor
3. **Review:** 
   - If inference result with yellow box → press Enter to confirm, or adjust with keys
   - If original with no box → draw box with mouse/keys or mark as no stamp
   - If uncertain → mark as "Skip" to defer
4. **Action saved:** Box coordinates and action saved to corrections_queue.json
5. **Next file:** Auto-advance or click "Next" button
6. **Train trigger:** Click "Train Now" when ready → executes train.py

## Data Flow

### On Startup

1. Scan `photo_mapping_samples/` → add originals to queue (status="pending")
2. Scan `infer_output/` → add inference results to queue (status="pending")
3. Load `corrections_meta.json` → populate inference_data for each file
4. Check `dataset/labels/` and `dataset/corrections/` → update status="reviewed" for already-labeled files
5. Serve corrections_queue.json to frontend

### On User Action

- **Confirm:** Save user_correction with action="confirmed" → copy label to dataset/labels/
- **Edit:** Save adjusted box to user_correction → copy to dataset/labels/
- **Mark No Stamp:** Save action="no_stamp" → add to skipped.txt
- **Skip:** Save action="skipped" → leave status="pending" for re-inference
- **Train:** Call `python train.py` (auto-merges corrections into main dataset)

### After Training

1. train.py merges corrections/ into main dataset/labels/
2. Dashboard clears corrections/ directory
3. Resets all statuses to "pending" for next inference run
4. User should run infer.py to regenerate predictions

```
  corrections_meta.json         # Pre-computed predictions (from infer.py)
```

## Error Handling & Edge Cases

- **File deleted mid-session:** Remove from queue, alert user
- **Inference results stale:** Show warning, prompt re-inference
- **Training in progress:** Disable "Train Now" button, show progress bar
- **Concurrent edits:** Queue state is source of truth, refresh on each action
- **No predictions for inference image:** Show blank canvas, allow manual annotation
- **Skipped files:** Mark as "needs_fix" to indicate they should be re-inferred

## Workflow Summary

```
1. User runs: python infer.py (batch inference, generates corrections_meta.json)
2. User opens: python corrections_dashboard.py
3. Dashboard loads: corrections_queue.json (all files + metadata)
4. User reviews queue: filters/sorts, selects files to review
5. User edits: confirms, adjusts, or marks boxes in canvas
6. User trains: clicks "Train Now" button
7. train.py executes: merges corrections, retrains model
8. User repeats: run infer.py again, review new predictions
```

## Success Criteria

✅ All files (original + inference) in one persistent queue  
✅ Metadata visible (status, source, confidence, timestamps)  
✅ Full editing capabilities (confirm, edit, mark no stamp, skip)  
✅ Manual training trigger  
✅ State persists across sessions  
✅ Reuses existing train.py logic  
✅ Pre-computed inference (batch mode, not real-time)

## What This Does NOT Include

- Real-time inference (batch pre-computation only)
- Multi-class annotation (single class: date stamp)
- Cloud/GPU training (CPU-only, local)
- Automatic retraining (manual button only)
